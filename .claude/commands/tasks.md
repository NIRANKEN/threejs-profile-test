# /tasks: 計画から実行可能なタスクプール（tasks.md）を生成するコマンド

## 目的

`/plan` コマンドで作成された計画情報（`plan.md`）を、AIエージェントまたは人間が逐次実行可能な粒度の細かいタスクリストへと分解し、`tasks.md` として出力・管理します。

## 実行ワークフロー

1. **計画の読み込み**: `doc/plans/plan.md` を読み込み、実装に必要なステップを把握します。
2. **タスクの分解**: 各ステップを「1回のPRやコミットで完結できる」最小単位に分割します。
3. **メタデータの付与**:
   - **依存関係**: タスクBがタスクAを必要とする場合、その依存関係を明記します。
   - **並列マーカー**: `[P]` タスク同士が依存せず、マルチエージェントで並列実行可能なタスクには `[P]` マーカーを付けます。
   - **担当エージェント**: `[Frontend]`, `[Testing]` など、どの領域のエージェントが担当すべきかを提案します。
4. **出力と管理**: `tasks.md` （または `doc/plans/tasks.md`）を出力します。

## tasks.md のフォーマット（例）

```markdown
# 実装タスク

- [ ] Task 1: 基礎コンポーネントのセットアップ [Frontend]
- [ ] Task 2: 物理エンジンの統合 [Backend/Physics] (依存: Task 1)
- [ ] Task 3: ユニットテストの作成 [Testing] [P]
- [ ] Task 4: UIオーバレイの作成 [Frontend/Palette] [P]
```

## 注意事項

- マルチエージェントで並行作業させるGit Worktreesパターンを想定し、コンフリクトが起きない単位でファイル領域の所有権を分割してください。
