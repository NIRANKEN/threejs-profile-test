# /plan: 仕様から実装計画（plan.md）を策定するコマンド

## 目的
`/specify` コマンドで作成された（または既存の）仕様書 `spec.md` に基づき、具体的なアーキテクチャ、データモデル、技術選定を含む実装計画 `plan.md` を作成します。

## 実行ワークフロー

1. **仕様の読み込み**: 対象となる `spec.md` を読み込みます。
2. **アーキテクチャ設計**: 
   - コンポーネント構造の設計
   - ディレクトリ構成（R3Fの `Canvas` 内部と外部Reactツリーの切り分け等）
   - 状態管理パイプライン（`useFrame` を用いた直接更新による最適化設計など）
3. **技術要素の選定**: プロジェクトの標準スタック（React 19, R3F, Vite 6, Vitest等）に基づく技術の選定とライブラリの利用方針を決定します。
4. **計画書の出力**: `doc/plans/` 配下に `plan.md`（または `MULTI_AGENT_PLAN.md`）として出力します。

## 注意事項
- 特にR3F（3D環境）が関わる機能では、**「どの処理をReactのステートで行い、どの処理を `useFrame` を介してGPU直接命令にするか」** の境界線を明確に設計に含めてください。
- マルチエージェント（Architect, Backend, Frontend, Testing等）で実装を分担する場合は、エージェント間の依存関係や受け渡しデータフォーマット（API契約等）をプランに記載してください。
